On-Page SEO Checklist: Beginner guide 2026

After finishing keyword research for my SEO project, the next step was on-page SEO — going through the page itself and making sure every element actually supports the keyword I chose, instead of just sitting there as plain content.

Here's the checklist I worked through, and what each step meant in practice, not just in theory.

1. Title tag

The title tag is the clickable headline shown in Google's search results. I made sure my target keyword sits near the start of the title, kept it under 60 characters so it doesn't get cut off, and wrote it to actually sound worth clicking — not just keyword-stuffed.

2. Meta description

This is the short summary line under the title in search results. It doesn't directly affect rankings, but it affects whether someone clicks — so I wrote mine to clearly state what the page delivers, kept to around 150–160 characters.

3. One H1 heading

Every page should have exactly one H1, and it should contain the main keyword. I used to think more headings meant more SEO value — that's wrong. A page with multiple H1s confuses search engines about what the page is actually about.

4. Subheadings (H2, H3)

I broke the page into sections using H2 tags, one per checklist item. This isn't just for Google — it makes the page scannable for a real reader who's skimming to find the one section they need.

5. Natural keyword placement

My keyword appears in the first paragraph, in the H1, and in a couple of subheadings — not repeated in every sentence. Keyword stuffing was one of the first mistakes I learned to avoid; it reads unnaturally and search engines are built to detect and penalize it.

6. Content quality

Beyond keywords, the actual content needs to answer what someone searching that term wants to know. I checked the top-ranking pages for my keyword before writing, to understand what "good" looked like for this specific search intent.

7. Image optimization

Any image I add gets a descriptive file name and alt text related to the page topic, instead of a generic camera-file name. This helps accessibility and gives search engines extra context about the page.

8. Internal linking

I linked this post back to my Projects page, where the live SEO project itself is documented. Internal links help visitors move through the site and help search engines understand how pages relate to each other.

9. SEO-friendly URL

Keeping the file name/URL short and descriptive, matching the topic, rather than something like a random ID — this makes the page easier to understand at a glance, both for users and search engines.

10. Mobile-friendly and fast-loading

Since most searches happen on mobile, I checked this page on a phone screen specifically, and kept images compressed so the page loads quickly — page speed is itself a ranking factor, so a slow page works against everything else on this list.

What's next

With on-page SEO applied, the next stage of my project is technical SEO — indexing and monitoring through Search Console. I'll document that with real data once it's live, on my Projects page.
